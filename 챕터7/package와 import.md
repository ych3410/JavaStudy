# Package와 import

---

## 1.패키지

### 1.패키지(Package)

**패키지란? 클래스들의 묶을을 말한다** 패키지는 클래스 또는 인터페이스를 포함시킬 수 있으며, 서로 관련된 클래스들끼리 그룹 단위로 묶어 놓음으로써 클래스를 효율적으로 관리가 가능하다.

> **패키지의 장점**
>
> 1. 패키지 체계를 유지함으로써 다른 개발자가 개발한 클래스 라이브러리의 클래스와 이름이 충돌하는 것을 피할 수 있다.
> 2. 같은 이름의 클래스 일지라도 패키지 명으로 구분이 가능하다.

> **패키지**
>
> 1. 하나의 소스파일에는 첫 번째 문장으로 단 한 번의 패미지 선언만 허용된다.
> 2. 모든 클래스는 반드시 하나의 패키지에 속해야 한다.
> 3. 패키지는 점(.)을 구분자로 하여 계층구조로 구성할 수 있다.
> 4. 패키지는 물리적으로 클래스 파일(.class)를 포함하는 하나의 디렉토리이다.

### 2. 패키지의 선언

패키지를 선언하려면 클래스나 인터페이스의 소스파일(.java)의 맨 위에 다음과 같이 한 줄만 적어주면 된다.

~~~java
package 패키지명;
~~~

위 코드와 같이 패키지 선언문은 반드시 소스파일에서 주석과 공백을 제외한 첫번째 문장이어야 하며, 하나의 소스파일에 단 한번만 선언될 수 있다.

해당 소스파일에 포함된 모든 클래스나 인터페이스는 선언된 패키지에 속하게 된다. 패키지명은 대소문자를 모두 허용하지만, 클래스명과 쉽게 구분하기 위해 소문자를 이용한다.

그렇다면 아까도 말했듯 우리는 모든 클래스를 패키지에 포함시켜야한다고 했다 그러나 지금까지 한번도 패키지를 추가한적이 없다 그 이유는 생성자때처럼 컴파일러가 자동으로 이름없는 클래스를 만들어주기 떄문이다. 즉 우리가 패키지를 따로 정해준게 없다면 전부 이름없는 패키지에 속해있는 것이다.

## 2.import

### 1.import문?

소스코드를 작성할 때 다른 패키지의 클래스를 사용하려면 패키지명이 포함된 클래스 이름을 사용해야 한다. 이때 매번 패키지명을 추가하지 않고 **import문을 이용해 패키지를 미리 명시해주면 사용하고자 하는 클래스 이름에서 패키지 명이 생략된다.**

import문은 컴파일러가 컴파일을 할때 각 클래스들의 패키지를 알아내 모든 클래스이름 앞에 패키지명을 붙여 준다.

**이클립스는 ctrl + shift + o가 자동 import문을 써준다고 하는데 우리가 사용할 intelliJ에서는 빨간줄이 쳐진 클래스이름에 마우스 포인트를 올려 alt + enter를 할시 자동 입력된다**

### 2.import문의 선언

모든 소스파일(.java)에서 import문은  package문 다음에, 그리고 클래스 선언문 이전에 위치헤애 힌다. import는 한 소스파일에 여러개를 선언할 수 있다.

> 일반적인 소스파일(.java)의 구성은 다음의 순서로 되어 있다.
>
> 1. package문 
> 2. import문
> 3. 클래스 선언

import문은 어떻게 선언해야할까??

```java
import 패키지명.클래스명;			//패키지내의 클래스명에 해당하는 클래스만 import
import 패키지명.*;					//패키지내의 모든 클래스 import
```

클래스 이름이 아닌 *를 사용하여 전부 import를 해버린다면 컴파일러는 모든 클래스를 찾아야하는 수고를 더하지만 **코드 성능에는 차이가 없다** 

다음은 코드를 보자

```java
import java.util.Calendwe;
import java.util.Data;
import java.util.ArrayList;
```

위와 같이 같은 패키지인 util에서 여러 클래스를 import할시에 위와같은 코드보다

```java
import java.util.*;
```

 이 코드대로 구현하는것이 더 효율적일 것이다.

### 3. static import문

import문을 사용하면 클래스의 패키지명을 생략할 수 있는 것과 같이 static import문을 사용하면 static멤버를 호출할 때 클래스 이름을 생략할 수 있다. 특정 클래스의 static멤버를 자주 사용할 때 편리하다.

```java
import static java.lang.Integer.*;	//Integer 클래스의 모든 static매서드
import static java.lang.Math.random;	//Math.random()만. 괄호는 붙이지 않는다.
import static java.lang.System.out;	//System.out을 out만으로 참조가 가능하다.
```

만약 위와 같이 import하였을 경우 다음 코드와 같이 코드가 매우 간결해 지는것을 볼 수 있다.

```java
System.out.println(Math.random());	//=>out.println(random());로 줄일 수 있다.
```
